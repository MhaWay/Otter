# Mobile Build Setup

Complete guide for building Otter mobile app (iOS + Android) with native Rust FFI.

## Quick Start

### Android
```bash
cd flutter_app/android
cp local.properties.example local.properties  # Edit with your paths
cd ..
flutter build apk --release
```

### iOS
```bash
cd flutter_app/ios
pod install --repo-update
cd ..
flutter build ios --release
```

## Prerequisites (All Platforms)

### Rust
```bash
# Install Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env

# Verify
rustc --version && cargo --version
```

## Android Setup

### 1. Android SDK & NDK

**Option A: Via Android Studio (Recommended)**
- Open Android Studio
- Tools > SDK Manager
  - ✓ Android 14 (API 34)
  - ✓ Build Tools 34.x.x
  - ✓ NDK 25.2.9519653
  - ✓ CMake

**Option B: Via Command Line**
```bash
# Install sdkmanager
# https://developer.android.com/studio/command-line/sdkmanager

sdkmanager "platforms;android-34" \
  "build-tools;34.0.0" \
  "ndk;25.2.9519653" \
  "cmake;3.22.1"
```

### 2. Configure local.properties

```bash
cd flutter_app
cp android/local.properties.example android/local.properties
```

Edit `android/local.properties`:
```properties
# Path to Android SDK (usually ~/Android/Sdk on Linux/macOS)
sdk.dir=/Users/username/Android/Sdk
ndkVersion=25.2.9519653

# Optional: explicit NDK path
# android.ndkPath=/Users/username/Android/Sdk/ndk/25.2.9519653
```

### 3. Install Rust Android Targets

```bash
rustup target add aarch64-linux-android armv7-linux-androideabi x86_64-linux-android
cargo install cargo-ndk
```

Verify:
```bash
cargo ndk --version
rustup target list | grep android
```

### 4. Build

```bash
cd flutter_app

# Debug APK
flutter build apk --debug

# Release APK (optimized)
flutter build apk --release

# Split APKs per ABI (smaller downloads)
flutter build apk --release --split-per-abi
```

**Build Output:**
```
build/app/outputs/apk/release/
├── app-release.apk                    # Universal APK (~40MB)
└── (with --split-per-abi)
    ├── app-arm64-v8a-release.apk      # arm64 (~35MB)
    ├── app-armeabi-v7a-release.apk    # ARMv7 (~30MB)
    └── app-x86_64-release.apk         # x86_64 (~35MB)
```

### 5. Install on Device

```bash
flutter run                           # Build + install on connected device
adb install build/app/outputs/apk/release/app-release.apk
```

### Troubleshooting Android

**"Gradle wrapper not initialized"**
```bash
cd flutter_app
flutter clean
flutter pub get
flutter build apk --verbose
```

**"Cannot find gradle.properties"**
```bash
# Gradle files should be auto-generated by Flutter
flutter create --platforms android flutter_app
```

**"NDK not found / Invalid NDK path"**
```bash
# Check SDK Manager for NDK installation
# Or set explicit path:
echo "android.ndkPath=/path/to/ndk/25.2.9519653" >> android/local.properties
```

**"libotter_mobile.so not found"**
- FFI compilation failed during gradle build
- Check gradle output: `flutter build apk --verbose`
- Manually test: `cd /path/to/Otter && cargo ndk build -p otter-mobile --output flutter_app/android/app/src/main/jniLibs`

**"Target not installed aarch64-linux-android"**
```bash
rustup target add aarch64-linux-android armv7-linux-androideabi x86_64-linux-android
```

See detailed guide: [flutter_app/android/README.md](flutter_app/android/README.md)

## iOS Setup

### 1. Xcode & Command Line Tools

```bash
# Install Xcode from App Store, then:
xcode-select --install
sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer

# Verify
xcodebuild -version
```

### 2. Install iOS Rust Targets

```bash
rustup target add aarch64-apple-ios x86_64-apple-ios aarch64-apple-ios-sim
```

### 3. Install CocoaPods Dependencies

```bash
cd flutter_app/ios
pod install --repo-update
```

This automatically:
- Installs Flutter framework
- Compiles Rust otter-mobile library (via Podfile)
- Creates libotter_mobile.a static library

### 4. Build

```bash
cd flutter_app

# Simulator (debug)
flutter run

# Device (release)
flutter build ios --release

# Manually build in Xcode
open ios/Runner.xcworkspace
# Xcode: Select Target > Scheme > Run/Build
```

**Build Output:**
```
build/ios/iphoneos/
└── Runner.app                     # iOS app bundle
```

### 5. Install on Device

```bash
flutter run -d <device_name>
flutter install

# Or install from Xcode:
# Product > Run (Cmd+R) or Product > Archive
```

### Troubleshooting iOS

**"Pod install fails"**
```bash
cd flutter_app/ios
rm -rf Pods Podfile.lock
pod install --repo-update
```

**"Cannot find libotter_mobile.a"**
```bash
# Verify Rust compilation
cd /path/to/Otter
cargo build -p otter-mobile --target aarch64-apple-ios --release
ls -la target/aarch64-apple-ios/release/libotter_mobile.a
```

**"Linker error: Undefined symbol for architecture arm64"**
- In Xcode > Target > Build Phases > Link Binary
- Verify libotter_mobile.a is linked
- Check Build Settings > Linking > Other Linker Flags

**"Command 'rustup' not found"**
```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"
```

See detailed guide: [flutter_app/ios/README.md](flutter_app/ios/README.md)

## FFI Bindings

### Dart ↔ Rust Integration

The Flutter app can call Rust functions via FFI:

```dart
import 'dart:ffi';
import 'package:ffi/ffi.dart';

// Load Rust library (platform-specific)
final base = Platform.isAndroid
    ? DynamicLibrary.open("libotter_mobile.so")
    : DynamicLibrary.process();

// Define FFI function signature
typedef GenerateIdC = Pointer<Utf8> Function();
typedef GenerateIdDart = Pointer<Utf8> Function();

final generateId = base
    .lookup<NativeFunction<GenerateIdC>>("otter_mobile_generate_identity")
    .asFunction<GenerateIdDart>();

// Call Rust function
void testIdentity() {
  final idPtr = generateId();
  final idStr = idPtr.toDartString();
  print("Generated ID: $idStr");
  malloc.free(idPtr);
}
```

See [crates/otter-mobile/src/lib.rs](../../crates/otter-mobile/src/lib.rs) for exported C functions.

## Deployment

### Google Play Store

1. Create signed keystore:
```bash
keytool -genkey -v -keystore ~/upload-keystore.jks \
  -keyalg RSA -keysize 4096 -validity 10950 \
  -alias upload

# Password: (create one)
```

2. Create signing config:
```bash
cat > android/key.properties << EOF
storePassword=<your-password>
keyPassword=<your-password>
keyAlias=upload
storeFile=/home/username/upload-keystore.jks
EOF
```

3. Update android/app/build.gradle to use signing config

4. Build signed APK:
```bash
flutter build apk --release
# OR bundle (recommended for Play Store):
flutter build appbundle --release
```

5. Upload to Play Console

### Apple App Store

1. Create App ID via [Developer Account](https://developer.apple.com/account/)

2. Configure signing in Xcode:
```bash
open ios/Runner.xcworkspace
# Xcode > Target > Signing & Capabilities
#   ✓ Automatically manage signing
```

3. Build archive:
```bash
flutter build ios --release --obfuscate
# OR manually in Xcode:
# Product > Archive
```

4. Upload via App Store Connect or Xcode
